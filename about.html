Git ve GitHub Nedir?

Git

Nedir?

Git, bir sürüm kontrol sistemidir.
Kullanımı basittir.
Linux, BSD, Solaris, Darwin, Windows, Android ve diğer işletim sistemlerinde çalışır.
Kısaca anlatmak gerekirse yaptığımız projelerin adım adım versiyonlarının kopyalarını alabildiğimiz ve ihtiyaç halinde bu versiyonlara geri dönmemeizi sağlayan basit sistemdir

Neden ihtiyaç Duyarız?

Örnek vermek gerekirse bir projemiz olsun bu projemiz üstünde değişiklik yapcağımız zaman projemizin güvende olması amacıyla bir kopyasını oluşturup kopya üstünde değişiklik yapıp eğer çalışıyor ise bu değişiklikleri ana projeye geçiririz ama bunu yapmak meşakkatli olduğu için bunu daha kısa yoldan ve bir kopya oluşturmaya gerek kalmadan git sürüm kontrol sistemi ile yapabiliriz.

GitHub

Nedir?

GitHub, kalabalık bir ekibin sürüm kontrol sisteminde bir yazılım geliştirirken kullanabilecekleri internet tabanlı bir depolama servisidir.
GitHub servisi sayesinde pek çok kişiden oluşan kalabalık yazılım geliştirici ekipler bile aynı anda aynı proje üzerinde çalışabilir ve ayrı değişiklikler yapabilirler.

Neden İhtiyaç Duyarız?

Sürüm kontrol sisteminde (git) kayıt aldığımız versiyonları başka cihazlardan ulaşmak istediğimzde bize yardım eder. GitHub üstünden repository denilen sürüm geçmişlerinin ve dosyaların tutulduğu depolara projemiji göndererek uzak cihazlardan ulaşım sağlayabiliriz

Repository:

Remote ve Local olmak üzere ikiye ayrılır. Remote repolar herkese açık veya gizli olma özelliklerine sahiptirler git dosyamızı diğer cihazlardan ulaşım için remote repolara yüklememiz gerekir. Local repolar ise sadece kendi cihazımızda görebilceğimiz repositorylerdir

.gitignore:

git sürüm kontrol sisteminin işleme almamasını istediğimiz dosyaları içine kaydetmemiz gerekir ister repository oluştururken otomatik olarak bu dosyayı oluşturabiriz istersek de proje klasörümüze kendimiz oluşturabiliriz
